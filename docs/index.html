The Problem

Why We Built SmartQueue

Support teams drown in tickets every day — but existing tools treat every request the same. We wanted to build something that actually thinks.

🌊

Unpredictable Ticket Floods

Queue depth spikes without warning. Agents either over-staff at idle times or miss SLAs during surges — with no early-warning system.

🏷️

Manual Triage is Slow & Inconsistent

Humans spend 10–20 % of their shift deciding priority, category, and routing — decisions that a well-prompted LLM can make in under a second.

🔎

Agents Search for Answers in Isolation

When a P1 server is down, agents hunt through runbooks on their own. There's no assistant that already knows the ticket context and the relevant IT procedures.

📉

No Real-Time Visibility

Dashboards show counts, not insight. Managers can't tell at a glance whether to escalate, batch low-priority work, or reassign tier-1 load.

Architecture

How It Works

Three loosely-coupled services communicate over Redis. Every request flows through an AI triage layer before touching the queue.

Browser (React + nginx :3001) │ ├─ /api/v1/* ──▶ Go API Server (:8080) │ │ │ Redis 7 │ (sorted set + │ hash + list) │ │ │ Go Worker Pool │ (2 replicas × 5 goroutines) │ └─ /api/ai/* ──▶ Python AI Service (:8000) │ ┌───────────────┼───────────────┐ │ │ │ Classifier Recommender Bot (SSE) (LLaMA 3.3) (LLaMA 3.3) (LLaMA 3.3) │ │ │ └───────────────┴───────────────┘ │ Groq API │ In-memory BM25 (10 IT runbooks)
1

Submit & Classify

Agent types a ticket description. The Python service calls LLaMA 3.3 70B via Groq and returns category, priority (P1–P4), SLA deadline, tier routing, and tags in under a second.

2

Enqueue with Priority

The Go API stores the job in a Redis sorted set keyed by composite score (priority × timestamp). A dead-letter list catches exhausted retries. A sweeper goroutine rescues stale processing jobs every 30 s.

3

Worker Pool Processes Jobs

Up to 10 goroutines (configurable) dequeue atomically using a Redis Lua script. Job handlers run in parallel; failures decrement the retry counter and re-enqueue automatically.

4

AI Recommends Actions

The recommender analyses live queue state and emits up to 4 prioritised actions — escalate, batch, defer, or reassign — with explanations, so managers can act before an SLA breach.

5

Streaming Bot Assists Agents

Agents open the AI Bot tab, select a ticket, and chat. The bot receives ticket context plus BM25-retrieved runbook excerpts and streams token-by-token via Server-Sent Events. Session history is persisted in Redis.

Capabilities

What Makes It Different

Every feature was driven by a real pain point in support operations.

🎯

Zero-Shot Ticket Classification

LLaMA 3.3 70B assigns category, priority, tier, SLA hours, effort estimate, and keyword tags in a single API call. Keyword fallback keeps the app working if the LLM is unavailable.

AI · Groq
📊

Workload Recommendation Engine

Analyses live queue depth, SLA risk distribution, and age of oldest jobs to produce up to 4 actionable recommendations with urgency scores.

AI · Analytics
💬

Streaming AI Assistant

Real-time token-by-token streaming via Server-Sent Events. The bot gets full ticket context plus relevant IT runbook excerpts retrieved by BM25 search.

SSE · RAG
⚡

Priority Queue with Retries

Redis sorted set schedules P1–P4 jobs. Failed jobs decrement a retry counter and re-enqueue automatically. Exhausted jobs land in the dead-letter list and can be retried via API.

Go · Redis
⏱️

SLA Breach Risk Scoring

Every job carries an SLA deadline. A real-time calculator buckets jobs into ok / warning / at_risk / breached and surfaces them in the Queue Health dashboard.

Python · Real-time
🔒

Prompt Injection Guardrails

Regex-based detection blocks common injection patterns before every LLM call. Rate limiting (30 req/min per client) prevents abuse of the AI endpoints.

Security
🔁

Stale Job Recovery

A background sweeper goroutine runs every 30 s and re-enqueues any job stuck in "processing" for more than 5 minutes — no manual intervention required.

Go · Resilience
📡

Horizontal Scaling

Worker replicas and goroutine pool size are independently configurable. Kubernetes HPA scales worker pods on Redis queue depth via the Prometheus Adapter.

K8s · HPA
🎭

Demo Ticket Simulator

One click floods the queue with up to 50 pre-classified realistic support tickets — perfect for demonstrating SLA risk behaviour and recommendation triggers without manual data entry.

Demo · DX
